Как мы с Вами [выяснили](./0.%20Введение.md) система контроля версий отслеживает и хранит все изменения. Но эти изменения нужно где-то хранить и где-то отслеживать. Все это можно ограничить папкой  - корневая (самая верхняя) папка проекта. Там могут лежать другие папки и файлы (и все изменения в них отслеживаются).

Как раз таки такая папка (корневая) и называется репозиторием - то есть папка, в которой начала работу система контроля версий. 

Если привести пример: репозиторий - это чемодан. В нем лежит куча вещей (фотографии, записки, вещи) и специальный дневник, который отслеживает всю историю содержимого чемодана (кто, когда и зачем положил, заменил или вынул вещь).

Любую существующую папку можно сделать репозиторием. Для этого достаточно открыть ее в терминале (открыть папку, кликнуть правой кнопкой на пустом месте и выбрать ```Открыть в терминале``` или ```Open Git Bash here```):

![Выпадающее меню проводника](./Материалы/Выпадающее%20меню%20проводника.png)

и ввести там команду:

```cmd
git init
```

 Если репозиторий создан успешно, вы увидите сообщение: ```Initialized empty Git repository in <путь до вашей папки>```. Если в папке уже существовал репозиторий он инициализируется заново и вы увидите: ```Reinitialized existing Git repository in <путь до вашей папки>```.
