# Что такое система контроля версий?

Если брать умное определение, то **система контроля версий (VCS - Version Control System)** - это инструмент, который сохраняет историю изменений в файлах проекта, позволяя разработчикам отслеживать изменения и управлять версиями кода. Основная цель VCS - упростить и упорядочить работу над проектом.

Звучит не очень понятно, поэтому рассмотрим пример:  представь, что ты пишешь курсовую в Word. Ты делаешь сохранение:

- `курсовая.docx`
- `курсовая_сдал.docx`
- `курсовая_пересдал.docx`
- `курсовая_точно_последняя.docx`
- `курсовая_точно_точно_последняя.docx`

В итоге у тебя появляется путаница, нет понимания связанности и какой файл действительно является последним. Тут на помощь и приходит контроль версий.

**Контроль версий** - это как магический помощник, который сам помнит каждое сохранение и позволяет в любой момент вернуться к предыдущей версии. Тебе не нужно плодить файлы с разными именами. То есть у тебя появляется файл (а может даже и папка, или набор файлов и папок) который хранит редакции (версии), к которым ты можешь вернуться в любой момент.

## Основные задачи контроля версий

1. **Запоминает историю** - каждый раз, когда ты сохраняешь изменения, система делает «снимок» проекта
2. **Позволяет откатываться** - если что-то сломал, возвращаешься к рабочей версии
3. **Показывает кто и что менял** - можно увидеть, кто именно удалил эту строку и когда
4. **Помогает работать вместе** - несколько человек работают над одним проектом, и система аккуратно соединяет их изменения

## Программное обеспечение

Есть несколько реализаций данного подхода. Самый популярный - [GIT](https://ru.wikipedia.org/wiki/Git). Данная система будет рассмотрена в последующих уроках. Для воспроизведения примеров необходимо [установить](https://git-scm.com/install/windows) его себе на ПК. В процессе установки можно просто нажимать "Далее" - для начальных задач стандартные настройки подходят лучше всего.
