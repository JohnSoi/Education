Файлы в [репозитории](1.%20Репозиторий.md) могут находится в нескольких состояниях (по отношению к системе контроля версий):
1. Не отслеживаемые (файлы, за которыми система не следит)
2. Отслеживаемые (файлы, изменения которых учитываются и отслеживаются)
3. Игнорируемые (файлы, которые не должны отслеживаться (часто меняемые (логи) или чувствительные (пароли и конфиги) данные)).

Отслеживаемы файлы подразделяются на:
4. Новые (ранее не отслеживаемые или впервые добавленные)
5. Измененные (изменения в ранее отслеживаемом файле)
6. Удаленные (удаление ранее отслеживаемого файла)

Статус файлов в репозитории можно посмотреть командой:

```cmd
git status
```

Если файлов в репозитории нет или все изменения сохранены, команда скажет об этом. Но если у нас появляются отслеживаемые или не отслеживаемые файлы, то мы увидим примерно такой вывод:

![Не отслеживаемые файлы в git status](./Материалы/Не%20отслеживаемые%20файлы%20в%20git%20status.png)

По умолчанию система контроля версий не отслеживает файлы (новые или измененные) для того, что бы Вы получили полный контроль того, что нужно сохранить. Чтобы добавить файлы в отслеживаемые используется команда:

```cmd
git add .
```

Вместо точки можно указать название файла или папки для добавления. Точка добавляет все файлы текущей папки в отслеживаемые. Данная команда, в случае успеха, не дает никаких ответных сообщений - пугаться не нужно.

После этого, команда ```git status``` выдаст информацию, что у нас есть новый файл, готовый к сохранению изменений:

![Отслеживаемые файлы в git status](./Материалы/Отслеживаемые%20файлы%20в%20git%20status.png)

Если нужно сбросить состояние файлов, до изменений используется команда:
```
git reset --hard 
```
```--hard``` затирает изменения полностью (удалит созданные файлы, которые не были закоммичены). Если вам нужно файлы оставить (например созданные, но без коммита), данный префикс нужно убрать (файлы перейдут в состояния не отслеживаемые).
